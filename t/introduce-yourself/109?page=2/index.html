I am Sung M. Kim from New York City, the Big .
I have been mostly working with backend (.net, sql server) and but found a new love in frontend world and has been making a transition to a frontend role.

My choice of frontend view library has been React and been enjoying it quite much and just started learning about GraphQL, PWA (https://undraw.netlify.com/ - An Offline-first PWA app for searching & downloading Undraw illustrations), and Gatsby.

I found out about Frontity from a dev.to post by Reyes Martínez, and I found it interesting as I was manually trying to create a static version of my WordPress blog.

The challenge while trying to create a static version of WordPress site using Gatsby was that, it was the plugin required to fetch WordPress data and having to create a normalization in gatsby-nodes.js to turn the WordPress data to be consumable with GraphQL.

The most challenging part was embedded GitHub gists, which I had to call GitHub to “evaluate” the javascript file to get the actual gist content.

Gatsby has so much flexibility as a general framework, it can do quite much but sometimes I’d need some opiniated way to get the WordPress data without having to “massage” the data.

orballo 17 May 2019 08:29 32

Hi @sungmkim80! It’s nice to have you here!

Frontity is definitely more opinionated towards WP. We have one package called @frontity/wp-source (docs here) that will handle all the fetching from your WP site. You just need to setup your site apiUrl in the frontity.settings.js file in your Frontity project.

Then you’ll use something like actions.source.fetch("/category/nature/") from your theme in order to get the data. In case you want to have a look, here is our starter theme repo where you can see how it’s done.

We hope you enjoy Frontity! See ya!

EDIT: Also, thanks for sharing your experience with gatsby so we can understand better where developers struggle when building sites for WP with React
