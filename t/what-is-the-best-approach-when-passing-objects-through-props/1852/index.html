mburridge 28 May 2020 10:17 15

Hi @dejangeorgiev

As I said earlier, I referred this issue to our dev team. Here is what they replied:

It’s a known problem with the WordPress REST API.

If you get a taxonomy directly using the specific endpoint (e.g. https://test.frontity.org/wp-json/wp/v2/categories ) the description property is included.

However, if taxonomies are embedded when you are retrieving posts (e.g. site.local/wp-json/wp/v2/posts?_embed=true ) they would be missing some properties, among which are count , description , parent and meta .

As the wp-source package tries to minimize the amount of calls to the REST API, once an entity (like a taxonomy) has been populated in the state it wouldn’t fetched again when you run actions.source.fetch - even if some properties are missing.

You can override this behaviour though using the force option when calling actions.source.fetch :

actions.source.fetch("/category/example", { force: true });

mmczaplinski 8 June 2020 17:20 32

Hey @dejangeorgiev!

Your problem regarding the infinite loop in the taxonomies seem to be that you are calling const TaxonomyData = actions.source.fetch('/cuisine'); inside of the body of the react component.

Calling fetch() will update the state, so when it finishes, it will re-render the GetTaxonomy component with new data, causing the actions.source.fetch('/cuisine') to be executed again, and again…

What you want to do is wrap the actions.source.fetch() with useEffect(), something like:

(disclaimer: this is not tested)

const Component = () => {
  useEffect(() => {
    actions.source.fetch(taxonomyLink);
  }, [])

  const data = state.source.get(taxonomyLink);

  if (!data.isReady) return null;

  const taxonomy = state.source[data.taxonomy][data.id];

  return <div>{taxonomy.name}: {taxonomy.description}</div>
}

EDIT: Fixed the code to use data.taxonomy.
