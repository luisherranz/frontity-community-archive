Hi @cianespablo, thanks for your post and welcome to the community

Automatic aliasing is a feature of Zeit Now, not of Frontity. So I recommend you to ask the same question in their support forums: https://spectrum.chat/zeit

To avoid SEO problems with multiple domains make sure to always use canonical tags in your <head> pointing to the right domain. Take a special look at the section 6 of this article, “Canonicalize cross-domain duplicates”: https://moz.com/learn/seo/canonicalization

I just want to do a quick update in this topic.

@zhandosweb pointed out in here that plugins like Yoast automatically manage the canonical tags of your WP theme, so there’s probably a better solution to avoid duplicate content:

1. Use a plugin to password protect your front-end, like explained by @ShaGui here:

3 questions from a newbie

I found an plugin for wordpress named Password Protected which allows access to the REST API. This may be the solution you are looking for.

More info in this topic: 3 questions from a newbie

2. Use Redirection to redirect all your front-end traffic to your Frontity site:

3 questions from a newbie

Another viable option is to add 301 redirections with the Redirection plugin.

You have to

  1. Add new redirection.
  2. Choose the REGEX option.

  1. Use this source URL that matches any url that is not wp-admin , wp-json , wp-login …
^(\/(?!wp-(json|admin|login|signup|content|comments-post\.php|cron\.php)).*)
  1. Use your Frontity domain as Target URL, followed by $1 , which is the pathname and query:
https://mars.frontity.org$1

I’ve just tested it in https://test.frontity.io and works great (it is disabled now).

More info in this topic: 3 questions from a newbie

Finally, we will add options to do this in our upcoming PHP plugin:

3 questions from a newbie

We are already thinking how to solve this, among other things, with our PHP theme.

It’ll have some modes:

  • Display a warning with a link to the Frontity site.
    Like Nude, but without including the canonical tags of SEO plugins.
  • 301 redirects to the Frontity site.
    Like the solution I posted using Redirection .
  • PHP Theme Bridge.
    Instead of being a PHP theme per se, it simply does a HTTP request to the Frontity server to get the final HTML, then it echos that HTML as if it were generated by PHP.

So there will be an official solution (or more than one) for this problem
