It seems to be a problem with the implementation we did to deprecate the URL constructor that we exported from frontity:

github.com

frontity/frontity/blob/2bdf7dbe44fec3b0c08aae7175bee76d47811a41/packages/frontity/src/utils/url.ts

/**
 * DEPRECATED.
 *
 * To be removed in the v2 version. Now that we've deprecated Node 8,
 * maintaining this is no longer required.
 *
 * It was meant to be used for compatibility with Node 8, where the global
 * `URL` was not present and therefore this was not possible without importing
 * URL from `"url"`.
 *
 * @example
 * ```js
 * const url = new URL("https://domain.com");
 * ```
 */

import { warn } from "@frontity/error";

/**
 * Re-export to add a deprecation warning.

For some reason, it doesn’t work well in Safari (tested in version 13.1.2) and it throws an error, making the code to fail in the init() action of @frontity/wp-source and showing the error you mentioned.

I opened an issue in order to fix this (frontity/frontity#565). In the meantime, it should work fine using the previous version of frontity (1.10.1).

We’ll keep you posted!

EDIT: We already find a solution and we are going to publish a new version today! You can wait for the new version of frontity instead of installing the previous one
