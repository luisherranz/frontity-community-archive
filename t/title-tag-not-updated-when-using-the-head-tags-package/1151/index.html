David 9 July 2020 13:35 23

Hi again, @dejangeorgiev

What the REST API - Head Tags plugin does is to simply parse the HTML the wp_head action generates for each entity and adds the result to the head_tags fields. If the title tag doesn’t appear, that means it wasn’t gererated during the execution of wp_head action.

Regarding how to add the <title> tag, there are two types of WordPress themes:

  • Those which let WordPress to add it automatically
  • Those which add it themselves

To let WordPress add the <title> tag, a theme should execute the following code:

function theme_slug_setup() {
  add_theme_support( 'title-tag' );
}
add_action( 'after_setup_theme', 'theme_slug_setup' );

That way, WordPress will render the title when the wp_head action is called. The function in charge of doing so is _wp_render_title_tag (defined in wp-includes/general-template.php) which is added to the wp_head hook in the wp-includes/default-filters.php file.

That’s the ideal solution, but if you don’t want WordPress to add the title automatically, the theme would have to add it somewhere (in functions.php, for example):

function theme_slug_render_title() {
  echo '<title>' . wp_title( '|', true, 'right' ) . '</title>' . "\n";
}

add_action( 'wp_head', 'theme_slug_render_title' );

Any of these options should work.

Try it out and let us know.

PS: Do not forget to clear the plugin cache right after!
