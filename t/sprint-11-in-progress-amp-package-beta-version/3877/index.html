Sprint 11 Planning

This sprint will go from January 26th to February 24th. This is going to be our focus:

Finish the WIP issues:

  • Finish the Infinite Scroll version 1.
  • BrowserStack - Cypress tests.
  • BrowserStack - Selenium tests.
  • Research - Server Extensibility.
  • Migrate Frontity Connect to RES.

High priority bugs

  • actions.source.fetch fails with links with query params for certain handlers.
  • REST API - Head Tags doesn’t work with AIOSEOP ^4.0.0.

AMP package - beta version

Based on this implementation proposal:

  • Move all the CSS to the head
  • Expose the HTML template, the render function and the App in libraries.
  • Overwrite the HTML template.
  • Change the render to be renderToStaticMarkup.
  • Remove /amp from link in actions.source.fetch and state.source.get.
  • Create minimum amount of AMP processors.

It isn’t included here but we want to start analyzing which functionalities we should include for a Consent Management Platforms solution and open a proper Feature Discussion. There is a lot of uncertainty on this regard so we have to investigate it first.

Apart from this, if we still have time, we would like to start a conversation about the best way to approach a fallback for old browsers where only Server Side Rendering is available (IE).

Sprint 11 Review

You can check the Sprint board for more details, but here you have a summary of what we have accomplished.

  • We closed the Server Extensibility research.
  • We addressed the priority: high bugs.
    • actions.source.fetch fails with links with query params for certain handlers was closed.
    • REST API - Head Tags doesn’t work with AIOSEOP ^4.0.0 - we contacted them AIO team and they will add a fix that will help us a lot on this regard.
  • We are really close to release an initial version of the AMP package to start testing it → AMP board.
  • Expose head and scripts API on frontity namespace.

There are some issues we have been working on but they are still in progress. This is the status:

  • Infinite Scroll hooks: David wrote down the Final Implementation and it’s just pending a final review.
  • AMP initial processors: The basics processors are created and he is working on the more complex ones that will need e2e tests.
  • Selenium tests: As explained here, most of the work is already done. We have to clean the whole branch and ensure everything works as expected.
  • BrowserStack Cypress tests: This still needs a deeper look.
  • Migrate Frontity Connect to RES: We are still analyzing the best approach for this.
