luisherranz 18 June 2021 07:43 10

Ok, I found the bug. It seems like Vercel started sorting the queries in their cache, so now if you visit /?b=b&a=a&c=c it will cache the result and it will use that HTML file for other URLs, like /?a=a&b=b&c=c. This is confusing Frontity because the URLs are not the same, and we do our own query sorting, which gets mixed up.

Anyway, I’ll see how it can be fixed
