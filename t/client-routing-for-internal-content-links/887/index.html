  ],

From an SEO perspective, I believe using absolute is the easiest to maintain.

There are arguments for both, and John Mueller (Google Search Guru at Google) says both works “if done correctly.”

However, if you don’t have the time or resources to focus on SEO, it’s easier to just make it all absolute for a set-it-and-leave-it point of view. That way you don’t need to worry about SEO and duplicate content etc.

luisherranz 4 January 2021 17:11 21
francis:

Would it make sense to write a processor to look for your current URL

Sure. It should be easy to do a processor for that. Something like this:

const relativeToAbsoluteLinks = {
  test: ({ node }) =>
    node.component === "a" &&
    node.props?.href.startsWith("/")
  processor: ({ state, node }) => {
    // Add `state.frontity.url` in front of all relative links.
    node.props.href = state.frontity.url + node.props.href
    return node;
  },
};
